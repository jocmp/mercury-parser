OpenVPN 2.7 Released with Support for DCO Linux Kernel Module, mbedTLS 4

This release also adds TLS 1.3 support with bleeding-edge mbedTLS versions and multi-socket support for servers.
by Marcus NestorFebruary 11th, 2026February 11th, 20260 Comments
Share this article:

OpenVPN 2.7 has been released today as the latest version of this virtual private network (VPN) system that implements techniques to create secure point-to-point or site-to-site connections.

Highlights of OpenVPN 2.7 include support for the new upstream DCO Linux kernel module, which will be available in future upstream kernel releases, multi-socket support to handle multiple addresses/ports/protocols within one server, mbedTLS 4 support, and TLS 1.3 support with bleeding-edge mbedTLS versions.

This release also introduces client-side support for the new PUSH_UPDATE control-channel message, which allows servers to send updates to options like routing and DNS config without triggering a reconnect, PUSH_UPDATE server support with new management interface commands to send PUSH_UPDATE option updates.

Two new environment variables have been introduced to communicate desired default gateway redirection to plugins like NetworkManager. Also, the ‚ÄúRecursive Routing‚Äù check is now more granular to only drop packets-in-tunnel if the destination IP, protocol, and port matches with those needed to reach the VPN server.

Among other changes, OpenVPN 2.7 improves client support for DNS options by including client implementations for Linux, BSD, and macOS systems with the default install and introducing a new client implementation for Windows systems to add support for features like split DNS and DNSSEC.

On top of that, this release brings architectural improvements on Windows by reinforcing the block-local flag with WFP filters, add support for server mode in the win-dco driver, generate Windows network adapters on demand, and run the Windows automatic service as an unprivileged user.

Data channel was improved as well by enforcing AES-GCM usage limit, as well as epoch data keys and packet format. Check out the release notes for more details about the changes included in OpenVPN 2.7, which is available for download as community-maintained Linux packages from here.
